# Checklist de Release – SalUD / Cliente-Servidor

## 1. Calidad básica
- Lint sin errores en frontend y backend.
- Tests automatizados pasan en frontend y backend.
- Cobertura mínima alcanzada en módulos críticos.
- Reporte de cobertura actualizado.

## 2. Flujos críticos
- Registro de usuario validado.
- Login validado.
- Crear cita validado.
- Reagendar/cancelar cita validado.
- Consulta de historial clínico validado.

## 3. API
- Colección Postman ejecutada en entorno de QA.
- Sin errores en endpoints críticos.

## 4. No funcionales básicas
- Respuestas dentro de tiempos aceptables.
- Validación de errores consistente.
- Endpoint `/health` responde OK.

## 5. Entorno y despliegue
- Variables de entorno revisadas.
- Migraciones aplicadas.
- Seed base aplicado si corresponde.

## 6. Entregables
- Plan QA actualizado.
- Evidencia de pruebas disponible.
- Versión/tag de release registrada.
